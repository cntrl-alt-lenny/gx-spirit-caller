**Suggested approach:**

1. **Pick the anchor.** Run
   `python tools/find_pattern_clusters.py --version eur main 0x020498f0`
   for the per-anchor view: it lists all 21 matched siblings + 148
   unmatched. Pick a matched sibling (any) as the propagation
   template — its `.c` file is on disk already.
2. **Match `func_020498f0`** (the anchor) by hand from the template
   shape. This is a one-function step, similar to brief 007's
   `func_ov000_021ac550` flow: write the C, build, objdiff to 100%,
   ship.
3. **Run propagation.**

   ```
   python tools/propagate_template.py apply \
     --template src/main/<matched-sibling>.c \
     --cluster main 0x020498f0 \
     --version eur
   ```

   The tool validates reloc shape per target before emitting; it
   refuses with `TemplateMismatch` on any sibling whose actual
   relocation pattern diverges, so accidental wrong-C ships are
   blocked at the tool level.
4. **Build + objdiff.** Run `ninja rom`, `ninja objdiff`, `ninja
   report`. Confirm every newly-generated file hits 100%. If any
   sibling's report shows <100%, drop it from the PR (don't
   manually patch — that defeats the cluster's "they're really the
   same shape" guarantee). Re-run the analyzer to confirm tier
   counts moved.
5. **Delinks blocks.** Use `tools/scaffold_batch.py --apply-delinks`
   for any new `complete` entries needed in
   `config/eur/arm9/delinks.txt`.

**Scope:**
