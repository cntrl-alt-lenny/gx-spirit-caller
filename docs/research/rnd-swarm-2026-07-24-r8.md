## strategy

**S — The tractable-C ceiling (29.97% EUR) is a two-module policy artifact, not a technical limit.** `progress.py` hard-codes `ASYMPTOTIC_MODULES={main,ov002}` at a frozen 10% headroom fraction, and those two modules are 78.3% of the .text denominator — so the entire published ceiling rests on one un-derived assumption over two modules. The label is self-refuting: main is the campaign's #1 EUR mover (+33,832 C-bytes this window, ~8x the next module) yet is tagged asymptotic, having consumed ~49% of its model-assigned asymptotic headroom in a single month; at current pace it surpasses its own static published ceiling in ~2 months. Reclassifying main asymptotic→finishable (75%) lifts the total ceiling from 715,125 B (29.97%) to 1,142,354 B (47.88%) — arithmetic independently reverified byte-exact. ov002's fraction is a priority/policy dial (byte-complete, 100% code%), not a wall, and should be reported separately. *Action:* run the built-but-unrun main small-tier resweep, then replace the hard-coded 0.10 with main's measured empirical ship-rate and publish a measured per-module ceiling. *Effort:* days.

**A — Near-miss auto-localizer: attribute each diverging byte to its C source line via the compiler's own DWARF `.debug_line`.** The near-miss loop reports one fuzzy % per function and agents then eyeball a manual capstone instruction-diff to guess which C construct is wrong — yet the exact source line for every diverging instruction already sits unused in each scratch object. `configure.py:140 -sym on` emits genuine DWARF `.debug_line` (empirically decoded: offset 0x0→line 27, 0x30→line 41, verbatim), and `fastmatch.py:445` already returns the diverging word-index; nothing joins them. **B — Oracle-driven mutation prober: property-test source forms against byte-identity to auto-derive the fixing lever.** Levers are discovered by hand-enumerated variant sweeps one brief at a time (`lever-payoff.md` is explicitly "a lookup table, not a proposal for new experiments"); no mutation/property-test tool exists in `tools/`. The byte-identical oracle plus fastmatch's fast single-TU compile make this mechanizable: given the localizer's flagged line, apply a small transform library (ternary↔if-else, operand swap, cast form, C89 hoist, unsigned-cast-for-blo) and let the oracle report which single mutation flips the diff to a match — byte-neutral mutations become proven safe-readability rewrites, byte-sensitive ones auto-populate the lever catalog. Distinct from the DEAD permuter (assembly reg/schedule vs. a fuzzy score) — this searches source forms against the exact oracle, localizer-guided. Soft-depends on the localizer above but the core search stands alone. *Action:* build `tools/neutrality_probe.py` emitting a machine-readable (mutation→diff-delta) lever DB, auto-stop at 100%. *Effort:* days.

**Biggest win:** Fixing the ceiling artifact (S) roughly doubles the campaign's north-star number from ~30% to ~48% for a days-long tooling change — a strategic reframing that outranks any per-function tooling gain.
