## Project structure & progress — the honest read

**Where it stands.** The ROM byte-matches all three regions. The campaign is now a two-front mechanical drain — ~1,038 remaining runbook `.s` ships plus C-match ports, effectively all of it in `src/usa` and `src/jpn` — while EUR readable-C sits near 8%. progress-trajectory surfaced no surviving delta: the trajectory and ship counts are as the ledger records. The four internal findings that *did* survive reframe the endgame and expose accreted debt:

- **The permanent floor is tiny; the rest is contingent (residual-floor).** Only ~42 `.s` files carry `mcr/mrc/swi/msr/mrs` — the C-8 coprocessor/SWI class no C operator can express and no future oracle or LLM can ever convert (only inline-asm-in-a-`.c`-wrapper, which is cosmetic). Everything else on the "wall" list — reg-alloc, CSE, frame layout — is *compiler-contingent*: a better compiler model closes it. Floor permanence is not uniform; ~42 files are bedrock, the rest is a modeling problem.
- **The runway has an unused byte-exact primitive (cmatch-runway).** dsd v0.11.0 ships a working `sig new` / `sig apply --dry` that emits reloc-masked signatures and dry-runs matches read-only. Nobody has aimed it at our *own* matched functions — it can census byte-identical clones across the 24 overlays (catching families the note-derived worklist misses) and byte-identify USA/JPN twins (sidestepping `port_to_region.py`'s known EUR-renamed-callee mis-port).
- **The goal metric is subtly wrong (metrics-goal).** `c_code_bytes` sums `.text` for the numerator but `.text`+`.init` for the denominator, so it structurally caps at **99.907%** — the north-star number can *never* read 100% even at full decomp — and understates EUR C% (7.992 vs 8.085). Plus two latent defects (ext-leak across `_dsd_gap`, no `complete` gate) that bite the dev inner loop. All three dissolve by reusing the canonical `parse_delinks_file`.
- **Tooling has accreted debt (structure-architecture).** 93 flat `tools/*.py`, no package boundary: `parse_delinks_file`/`parse_symbols_file` exist but ~30 tools hand-roll the same parse, and 40/93 carry identical `sys.path.insert`+`# noqa: E402` boilerplate. Separately, a 3,990-file / 18 MB `docs/research/c-match-prep/` scaffold dump uses the `.c` extension, isn't search-excluded, and was never pruned — so agent Grep returns unverified scaffolds (276 already superseded by real matches).

**Highest-leverage next move: close the C-feedback loop.** The readable-C frontier — the *actual matching work*, and where essentially all churn happens — has zero automated compile/match feedback. Not in-loop: `post_edit.py` returns early on any non-`.py` path, so editing `src/**/*.c` yields nothing and the agent must remember `ninja`+`ninja objdiff` by hand. Not in CI: all 11 workflows are metadata/lint/python-only — nothing compiles a single game `.c`, even though the compile step is provably baserom-free and therefore CI-able. Two cheap fixes (**S1** hours, **S2** a day) close both halves and protect exactly the lane the whole campaign now runs on. Do these before any lever/permuter/oracle work — those chase the *contingent* walls, which the residual-floor result says are a modeling problem better handed to the Mac/oracle lane, not the bottleneck for the mechanical frontier.

## Top improvements (ranked)

### S-tier

**S1 — In-loop per-`.c` compile + objdiff gate in `post_edit.py`.** The PostToolUse hook fires on every Edit/Write but returns 0 for any non-`.py` file and any path outside `tools/`/`tests/` (`if rel.suffix != '.py': return 0`), so `src/**/*.c` edits — the live cmatch lane across all 5 worktrees — get no feedback. CI cannot fill this (baserom not redistributable); local is the only feedback site. mwccarm compiles offline, the per-TU `ninja build/<ver>/<path>.o` target exists, and `objdiff-cli` emits machine-readable per-function match% for a single unit. **Action:** extend the hook so `src/**/*.{c,cpp,s}` edits run a non-blocking per-TU compile, then single-unit objdiff on success, surfacing compile errors + unit match% to stderr with the existing exit-1 warning pattern (detect active region from build.ninja; ~1–3 s, non-blocking). **Effort: hours.**

**S2 — Baserom-free compile-only CI gate.** A GitHub Actions job can compile changed game `.c` to ELF with zero baserom — proven: `src/jpn/main/func_02000c44.c` compiles to a valid ARM relocatable using only committed `include/`,`libs/` + `mwccarm.exe` (baseroms untouched); 0 mwcc edges reference `extract/`; `configure.py <ver> --skip-sha1` already runs baserom-free in CI. Today nothing in CI compiles game C — the workflow comments conflate "can't verify byte-identity" with "can't build at all." **Action:** add `.github/workflows/compile-check.yml` (PR, paths `src/**`): fetch mwccarm(+wibo), `configure.py <region> --skip-sha1`, map changed `.c` via `git diff` → `build/<region>/<path>.o`, `ninja` those targets. Changed-file scoping keeps it seconds; **windows-latest needs no wibo at all**, eliminating the only Linux risk. **Effort: ~1 day.**

### A-tier

**A1 — `dsd sig` internal clone-census + region-port automation.** `sig new -c <cfg> -f <fn>` emits a reloc-masked byte signature read-only (validated: `VBlankIntrWait` → `pattern ACIF33BH`, git clean), and `sig apply -s <sig> --dry` dry-runs matching. Verified from a 3-region worktree: an EUR sig `--dry`-applied to `config/usa/arm9/config.yaml` finds the USA twin, and reloc-masking correctly zeroes the 7 reloc offsets of a reloc-heavy overlay fn; one config censuses main + all overlays. **Action:** prototype `tools/sig_census.py` — `sig new` → `sig apply --dry` across overlays (clone census for the *active* lane, diffed against the heuristic worklist to catch misses) and against USA/JPN (exact-twin map + auto-named ports for the *deferred* region endgame, bypassing `port_to_region.py`'s EUR-rename mis-port). Run from a worktree with `extract/{eur,usa,jpn}`. Yield is empirical — the prototype measures it; the primitive is proven. **Effort: days.** *(Extends r3's external `sig new-elf` lead to our own sources.)*

**A2 — Region-matrix the metadata CI gates to USA/JPN.** Every gating workflow runs `--version eur` exclusively, so USA/JPN — where all current churn lives — get zero metadata validation. Per-region runs today: EUR err 3/warn 11703, USA err 3/warn ≈11301, JPN err 3/warn 11282 — real state CI never inspects for two of three regions. A `missing_tu_source` carved into `config/usa/**/delinks.txt` with no `.c` passes CI green. **Action:** convert the invariants + analyzer PR jobs to `strategy.matrix.version: [eur,usa,jpn]` (one-line, identical script); gate usa/jpn advisory-only first (the 3 standing global name-drift errors fire in every region), promote to blocking once baselined. **Effort: hours.**

**A3 — Fix the goal metric's section-set mismatch (the 99.907% ceiling).** `c_code_bytes` sums `.text` for the numerator but `.text`+`.init` for the denominator; EUR's 2,224 `.init` bytes are owned by real `.c` TUs (e.g. `sinit_ov002_*.c`) that can never enter the numerator, so C% understates and structurally caps at 99.907% — the metric driving the whole campaign cannot reach 100%. Plus an ext-leak across `_dsd_gap` (latent) and no `complete` gate (inflates the dev inner loop). **Action:** replace the bespoke regex scan with the canonical `parse_delinks_file`, aligning numerator+denominator on `.text` (drops sinit/static-init glue from the C denominator); add a `status=='complete'` variant + the two missing tests. This is the acute instance of the parser-duplication debt (B-tier below) — fix it here first since it touches the goal number. **Effort: hours.**
