# Brief 296 — ov002 0x100-0x200 tier pre-map + core-header extension

**Status:** mapped + header extended. decomp.me / research / tools only —
no SHA1, no `.c`/`.s` shipped (`ov002_core.h` is a research artifact under
`docs/research/`). Fetched content as data. **Headline: the 0x100-0x200
band (695 funcs / 244 KB) is 58 % reachable (404 non-loop) / 42 % loop-
walled, and it largely REUSES the `<0x100` family vocabulary — same hub
`021ae400`, same `cf16c` accessor core, same `021d479c`/`021ca2b8`/
`0229ade0` sinks. The band's catch: reachable members are COMPOSITE
(multi-sink orchestrators, no pure wrappers), so the lever is shared
header VOCABULARY, not template-clone. `ov002_core.h` is extended with the
band's new top sinks (mostly ground-truth from matched `.c`) so the
decomper's velocity holds — but each member is per-function hand-RE.**

## (A) Shape census — 58 % reachable, dispatcher-heavy

`size_census.py --module ov002 --shape` over the current-unmatched band:

| class | funcs | bytes |
|---|---:|---:|
| simple | 91 | 28,668 |
| dispatcher | 313 | 113,288 |
| **reachable (simple+dispatcher)** | **404 (58 %)** | **141,956 (58 %)** |
| loop (reg-alloc-walled) | 291 (42 %) | 101,552 |

Versus `<0x100` (brief 284: ~69 % reachable, simple-dominated), the band is
**dispatcher-heavy** (313 dispatchers vs 91 simple) and **more loop-walled**
(42 % vs ~31 %) — bigger functions branch and loop more. The 291 loops go to
the brief-292/294 tail (hand-drain the ~10 % simple-leaf, ship the rest
as `.s`).

## (B) Families + worklist — reused vocabulary, composite bodies

Clustering the 404 reachable by shared callee shows the band leans on the
SAME sinks as `<0x100`, plus a handful of new top ones:

| sink (callers in band) | role | in header? |
|---|---:|---|
| `021ae400` (84) | hub: flag-gate then post a command | NEW [shipped] |
| `022593f4` (49) | d016c-state boolean query | NEW [shipped] |
| `0223df38`/`0223de94` (38/31) | node / VBits accessor | NEW (`de94` [shipped]) |
| `021e2c5c`/`021e2b3c` (36/35) | game-state tickers | NEW [shipped] |
| `021e286c` (33) | d016c-field → `021d479c` arg-pack | NEW [shipped] |
| `0225368c` (32) | vtable-dispatch (`blx` through fn-ptr table) | NEW |
| `021d479c` (27), `021ca2b8` (24), `0229ade0` (18) | reused `<0x100` sinks | in header |
| `021c1ef0`/`021c1e44` (band callers) | 0x868 accessors at band size | in header |

**The catch — composite, not batch-clone.** Unlike `<0x100` (which had
~44 pure single-sink wrappers), the band has **zero** simple ≤2-sink
wrappers — every reachable member calls 3-7 sinks and orchestrates them
with multi-guard control flow. Example: the **extended-accessor family**
(`021ea1b4`, `021eb530`, `021e75a8`, `021ee704`, … — band-sized callers of
`021c1ef0`) shares one recipe — `021b8fcc` guard → `Ov002Slot.id` guard →
`cf1a4`/`cf1ac` reads → an intricate guard-5 bitfield compare
(`2*((word>>22)&0xff) + ((word>>13)&1) == (self->f4>>6)&0x1ff`) →
`021c1ef0` → `021c2084` — but each member also carries per-function
reg-alloc and bitfield-typing work (a band member compiles structurally
correct from the recipe yet needs the same per-function tuning the brief-294
wall describes, just below the loop threshold).

**So the header's value here is shared TYPES + sink signatures** (the
decomper stops re-deriving `021ae400`'s flag-gate, the `cf16c`/`Ov002Slot`
layout, the arg-pack sigs), turning each band member from "reverse the
vocabulary + write the body" into "write the body" — but it is **not** a
clone-and-rename batch. Drain small-first within each family.

**Ranked worklist** (reachable, by leverage):

1. **Reused `<0x100` sinks** (`021d479c`/`021ca2b8`/`0229ade0`/`021c1ef0`
   /`021c1e44`, ~90 band callers): header already covers — start here, the
   recipe is proven.
2. **Hub `021ae400` family** (84): one-line flag-gate sink; callers are
   command-posters — composite but the sink is trivial.
3. **Extended-accessor family** (`021ea1b4` cluster): the brief-282/286
   recipe + guard-5 bitfield; batch the recipe, tune per-member.
4. **State/query sinks** (`022593f4`, `021e286c`, `021e2c5c`/`2b3c`,
   `0223de94`/`df38`): header sigs added; per-function bodies.
5. **`0225368c` vtable-dispatch** (32) + the loop tail (291) → lower-EV /
   `.s`.

## (C) Header extension — `docs/research/ov002_core.h`

Added a "0x100-0x200 BAND" section: the new top sinks + globals the band
needs. Provenance:

- **Ground-truth `[shipped]`** (signature read from a matched `.c` on main
  — objdiff-verified by being shipped byte-identical, per the brief-286
  convention): `021ae400(int,int)`, `022593f4(void)→int`,
  `021e286c(int)→int`, `021e2c5c(void)`, `021e2b3c(void)`,
  `0223de94(void*,int)→u16`, `021b1570(int,int,u16*,int)`.
- **New globals**: `data_ov002_022cd744[]` (flag table), `…ce720[]` (node
  ring); + a documented note on the `d016c`/`ce288` per-player state-struct
  field offsets the tickers read.
- **Recovered `[member]`** (signature from disasm call-sites, the
  pass-through accessor recipe): `021b8fcc(player,idx)→int`,
  `021c2084(self,player,idx,d)→int`.

**Verification.** The seven `[shipped]` sinks are ground-truth (matched on
main). The `021ea1b4` direct-mwcc pilot confirms the header's accessor
recipe is correctly reused at band size — the structure (guards, `Ov002Slot`,
`cf16c`/`cf1a4`, `021c1ef0`/`021c2084`) compiles as modelled — while
quantifying the band's per-function cost: the composite body needs reg-alloc
+ guard-5 bitfield tuning to reach byte-exact (it is not a first-try match
like the `<0x100` wrappers). No new pure-wrapper class exists to batch-verify;
the header delivers vocabulary, the decomper delivers bodies.

## Falsification

- "58 % reachable / dispatcher-heavy": `size_census.py --shape` filtered to
  the band — re-runnable.
- "reuses `<0x100` vocabulary": shared-callee scan — `021ae400`/`021d479c`/
  `021ca2b8`/`cf16c`/`ce288` dominate both bands.
- "composite, no pure wrappers": 0 band members with ≤2 sink calls (scan).
- "header sigs correct": the `[shipped]` sinks are matched `.c` on main;
  the `021ea1b4` recipe compiles structurally as modelled.
