## Struct context

`data_ov006_0224f3e0` — stride bitmask table descriptor. Word at `+0x2` (u16) is the
6-bit filter-enable flags register. Each bit enables one filter mode:

| bit | enable flag | query func | reference table | comparison mode |
|-----|-------------|------------|-----------------|-----------------|
| 0 | `#0x1` | `func_0202bbdc` | `data_ov006_0224f400[idx*7]` | exact s16 match |
| 1 | `#0x2` | `func_0202de9c` | `data_ov006_0224f402[idx*7]` | range (0xf0/0xf1 special) |
| 2 | `#0x4` | `func_0202b8a8` | `data_ov006_0224f404[idx*7]` | range classifier |
| 3 | `#0x8` | `func_0202b890` | `data_ov006_0224f406[idx*7]` | exact s16 match |
| 4 | `#0x10` | `func_0202b878` | `data_ov006_0224f408[idx*7]` | exact s16 match |
| 5 | `#0x20` | `func_0202de9c` + `func_0202b8c0` | `data_ov006_0224f40a[idx*7]` + `021cb310/021cb32c` | indirect table lookup |
| 6 | `#0x40` | `func_ov006_021b8d9c` | `data_ov006_0224f40c[idx*7]` | bitmask test (reuses 021b8d9c) |

The filter_idx stride is `*7` in ENTRIES (ldrsh with `mul r2, r5, r1` where `r1 = 0xe`
= 14 bytes = 7 × s16).

## Why still intractable

1. Six independent filter paths each with its own early-return — the reg-alloc
   across all 6 `ldrmneia` / `movne` return-zero patterns is permuter-class.
2. Bit 1 has `0xf0`/`0xf1` range sentinels not yet named in the KB.
3. Bit 2 has a three-way range classifier with `[1..4]`, `[5..6]`, `[7+]` thresholds
   not named as constants.
4. Bit 5's indirect double-dispatch via `data_ov006_021cb310`/`021cb32c` table lookup
   adds a nested comparison the permuter would need to solve.
5. 249 instructions total — far above the permuter's practical ceiling for convergence
   without macOS.
