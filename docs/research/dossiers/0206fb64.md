# 0206fb64 — func_0206fb64 (main, class D, 200B)

## Recipe + Risk
**Recipe:** D store-order/bitfield: gate on global!=0; BE byteswap (u16)(v>>16)>>8; ordered byte stores
**Risk:** the gate value v is loaded once (cmp), then RELOADED for each of bytes 2..5 (4x ldr [r1]); mwcc CSEs to one load -> bind/reload divergence; reshape needs separate re-reads, else permuter-class

## Closest matched example
**Path:** src/main/Task_PostLocked.c
**Key lesson:** This is the smaller sibling of `func_0206fa48` — same BE byteswap idiom with one TLV instead of two. The same CSE problem applies: orig loads `*data_0219ef00` once for the gate `cmp` (using r1 as the base register), then issues four additional `ldr [r1]` instructions inside the if-block, one per byte. mwcc -O4 CSEs all five loads to one. Adding `volatile` to the `v` declaration may force extra reloads but may also add too many; the exact count needs to match the orig's pattern of gate-load + 4-reload.
