**Why similar:** Same PPT stride pattern (mla r2,r1,0x868) with id_lo13 bitfield extraction (lsl#19/lsr#19) and slot-loop structure.
**Key lesson:** The id_lo13 bitfield must use the shift-pair form `(word << 19) >> 19` in C — not `& 0x1FFF`. The occupancy guard bit is a separate bitfield extracted with `(word << 10) >> 31`. mwcc emits both as paired lsl/lsr instructions; any deviation will shift registers.
