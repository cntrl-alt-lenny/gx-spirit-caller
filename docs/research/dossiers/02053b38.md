## Closest matched example

**Path:** src/main/func_02053ba4.c (see below — near-clone)
**Why similar:** `func_02053ba4` (linear search) uses the same dual-counter pattern and a `blx` through a function pointer; both functions are in the same source cluster and share the same register-allocation wall around the `blx`.
**Key lesson:** The `mla` instruction must be `mla r0, r4(mid), r9(stride), sl(base)` — mid-first, then stride, then base-as-accumulator. If the compiler generates `mla r0, r9, r4, sl` (stride-first) the encoded multiplication differs in register assignment. Declaring `lo` before `hi` in source drives `lo` into r6 and `hi` into r5, which is the observed register assignment in the original. The `blx` through the stack-passed fn ptr flushes caller-saved registers, causing a coin-flip in which of `lo`/`hi`/`mid` are saved to stack vs held in registers across the call.
