**Key lesson:** The inner-pointer bind pattern `b = p->_c` into a callee-saved register is the same shape. The critical divergence point is LICM: orig hoists both the address computations `&b->_2ec` / `&b->_3a4` AND the integer constants `0` and `1` into callee-saved registers (r7/r6) before the loop entry label, resulting in a 6-register `stmdb` prologue. If mwcc opts to rematerialize `mov r6, #0` / `mov r7, #1` inside each iteration instead of hoisting, the prologue register count drops and the binary diverges. No C-level reshape reliably forces LICM of integer constants.
