**Levers:**
- `mov r6, r0, asr r9` at entry (not `asr #r9`): mwcc emits a register-shift form for the initial shift; `shifted = kana_idx >> (bits * 3)` in C computes the same thing.
- `asr #0x1` per iteration: `shifted >>= 1` (signed shift right 1, preserving sign).
- `bics r0, r6, #0x1 / beq .L_64`: "has more bits beyond bit 0" test; reproduce as `(shifted & ~1) != 0` in the n_played==3 branch.

- `strb r9, [r0, #0x16c]`: stores the current `step_base` value (which equals the running se-step).
