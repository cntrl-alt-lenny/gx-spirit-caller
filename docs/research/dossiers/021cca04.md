## Struct context

| offset | field | type | use |
|--------|-------|------|-----|
| `+0x264` | camera_lock | `int` | if == 1: enable slot-5 extra check |
| `+0x268` | actor_facing | `int` | 4-bit slot = `(facing << 23) >> 28` |

`data_ov011_021d4000[0x2a0]` — view_mode word; 0 = scroll-slot 0.

`data_ov011_021d2fc8[slot*2]` — scroll target X for each actor slot (u8).
`data_ov011_021d2fc9[slot*2]` — scroll target Y for each actor slot (u8).

Note: `r7 = 0` throughout — the X upper clamp is `0 << 12 = 0`. `r8 = 0xe8` when the slot-5 SystemWork condition is met, else `0`. The Y upper clamp is `r8 << 12`.

## Recipe sketch

```c
void func_ov011_021cca04(void) {
    int  view = *(int *)(data_ov011_021d4000 + 0x2a0);
    int *act  = (int *)data_ov011_021d403c;
    unsigned char *lut_x = (unsigned char *)data_ov011_021d2fc8;
    unsigned char *lut_y = (unsigned char *)data_ov011_021d2fc9;

    int slot = 0;
    if (view != 0) {
        slot = (unsigned int)(act[0x268/4] << 23) >> 28;  /* 4-bit extract */
    }
    unsigned int tgt_x = lut_x[slot * 2];
    unsigned int tgt_y = lut_y[slot * 2];
    int extra = 0;

    if (act[0x264/4] == 1) {
        int slot2 = (unsigned int)(act[0x268/4] << 23) >> 28;
        if (slot2 == 5) {
            int *sw = GetSystemWork();
            /* bit 12 test: (sw[0x8f8/4] << 19) >> 31 == 0 */
            if (((unsigned int)(sw[0x8f8/4] << 19) >> 31) == 0)
                extra = 0xe8;
        }
    }

    /* clamp scroll_x to [(tgt_x*8 - 0x100) << 12, 0] */
    int cur_x = act[0x1fc/4];
    int lo_x  = ((int)tgt_x * 8 - 0x100) << 12;
    if (cur_x > lo_x) act[0x1fc/4] = lo_x;
    else if (cur_x < 0) act[0x1fc/4] = 0;

    /* clamp scroll_y to [(tgt_y*8 - 0xc0) << 12, extra << 12] */
    int cur_y = act[0x200/4];
    int lo_y  = ((int)tgt_y * 8 - 0xc0) << 12;
    if (cur_y > lo_y)            { act[0x200/4] = lo_y; return; }
    if (cur_y < (extra << 12))   act[0x200/4] = extra << 12;
}
```

**Levers:**
- `lsl #0x17 / lsr #0x1c` = 4-bit slot at bits 8–11 of actor_facing: write `(unsigned int)(act[0x268/4] << 23) >> 28`.
- The `ldrne` / `moveq` conditional in the view-mode check: `slot = (view != 0) ? extract(...) : 0`. Must NOT use a nested if; use the ternary form or `slot = 0; if (view) slot = ...;`.
- Y-clamp `ldmgtia` early return: when `cur_y > lo_y`, the function returns immediately after the strgt — write `if (cur_y > lo_y) { act[0x200/4] = lo_y; return; }`.
- `r7 = 0` is the X hi-bound: write `if (cur_x < 0) act[0x1fc/4] = 0;`.
- SystemWork bit-12 test: `(sw[0x8f8/4] << 19) >> 31` — the `lsl #0x13 / lsr #0x1f` chain; write as unsigned shifts on the int value.

## Try this

1. The X-clamp upper bound is `0` (r7=0 never changes) — write `if (cur_x < 0) act[0x1fc/4] = 0;`, not `if (cur_x < hi_x)`.
2. The Y early-return on `strgt`: only the Y path has `ldmgtia` (return-on-clamped). X does NOT early-return — after the strgt/strlt, execution always falls through to the Y clamp.
3. Slot extraction is done twice (once for the LUT index, once for the slot-5 special case inside the camera-lock check). Write two separate extractions, not a cached `int slot` reused.
4. The `movs r0, r0, lsr #0x1f` (sets Z flag) for the bit-12 test: write `if (... == 0) extra = 0xe8;` — the condition is `bit_is_clear` (moveq fires on zero result).
5. Park as .s if the `strgt/strlt` pattern for either clamp cannot be reproduced — some mwcc SP revisions emit a branch-based if/else instead of predicated stores for this shape.
