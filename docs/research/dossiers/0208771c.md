**Key lesson:** The `movs r4,r0` / `bne` / `beq` pattern for a return-value null-check requires that the first call's result be the input to the `if (node == 0)` test with no intervening operations. Keeping `node` as the first local in the function body drives mwcc to assign it to r4 (the callee-saved register that survives the second cfdc call). The signed comparison `arg < node[0x3d]` requires that node[0x3d] be read as a `signed char` (ldrb + sign-comparison against arg); if `arg` is declared `int` and node[0x3d] is cast directly, the compiler uses a `cmp r5,r1` with `bge` — the operand order matters.
