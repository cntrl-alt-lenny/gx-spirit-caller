**Why similar:** sibling function that clears bits 24..31 (top byte) of `sys+0x8e4` via `bic r1,r1,#0xff000000` — same word, same mask, simpler operation.
**Key lesson:** mwcc emits `bic` (not `and ~mask`) when the negated mask fits a rotated-immediate. `0xff000000` is `0xff rotated by 0` (top byte), and `0xff0000` is `0xff rotated by 16` — both are ARM rotated immediates, so both emit as single `bic` instructions.
