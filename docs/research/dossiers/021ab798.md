**Levers:**
- `smull r0, r6, r1, r2` — magic is in `r1`, value in `r2`; argument order to `smull` matters for mwcc.
- `add r6, r2, r6` after smull — mwcc's unsigned-reciprocal fixup; must be present as a separate add.
- `asr #0x7` on hi-word — 7-bit arithmetic shift after the fixup gives /255.
- The `mvneq r0, #0x0 / movne r0, #0x1 / lsl #0x4` triple is the mode_b→multiplier chain; use an explicit `int mul = (mode_b == 0) ? -16 : 16` rather than nested ternaries.
- `rsbne r1, r5, #0xff` — mwcc emits `rsb` for `255 - x` when `x` is in a register; writing `(int)(0xff - alpha)` should reproduce it.
- `subs + movmi` = saturating subtract; use `if (new_alpha < 0) new_alpha = 0` (signed underflow check).
