## Re-triage

**Original verdict:** low — "tile-lookup dispatch, nested mul/shift addressing, many live temps"

**New verdict:** HIGH — three-way dispatch on `data_ov011_021d4000[0x2a0]` (0, 1, other) with clean return paths. The `mul lr, lr, r0` + `ldrh` tilemap lookup is a standard row-major tile-fetch. The 3-bit stride extract `lsl #0x1d / lsr #0x1d` from `act[0x29c]` is confirmed by the matched `func_ov011_021ca324.c` which writes `*(int *)(s + 0x29C) & ~7`. Two paths (view=0 and view=1) differ only in the row-width shift (view=0 uses `lsl #0x5`, view=1 uses `lsl #0x6`). Literal constant `0x000003ff` (the mask for the tile index bits) is embedded as a `.word` literal pool entry.

**Why newly tractable:** `data_ov011_021d403c[0x29c]` = stride field (3-bit), `[0x290]` = tilemap pointer, `[0x28c]` = sub-tile table pointer are all accessible from the actor base. View modes 0 and 1 map to `OV011_VIEW_SCROLL` and `OV011_VIEW_SLOT` from OverlayConstants.md. The `and r0, r4, r0` with the `0x3ff` mask extracts tile index bits 0–9. The `ldr r0, _LIT2 / and r0, r4, r0` + `mov lr, lr, asr #0xc` pair decode the tile attribute sub-fields. No Mac dependency; the register usage is deterministic.

## Recipe sketch

```c
int func_ov011_021ca400(int x, int y) {
    int  view     = *(int *)(data_ov011_021d4000 + 0x2a0);
    int *act      = (int *)data_ov011_021d403c;
    int  stride   = (unsigned int)(act[0x29c/4] << 29) >> 29;  /* 3-bit */
    unsigned short *tilemap  = (unsigned short *)act[0x290/4];
    unsigned char  *subtable = (unsigned char  *)act[0x28c/4];

    if (view == 0) {
        /* 32-col tilemap */
        int col   = (y >> 3) * 32 + (x >> 3);
        unsigned short tile = tilemap[stride * col];
        int tile_idx  = tile & 0x3ff;
        int attr_nib  = (tile & 0xf000) >> 12;
        int sub_off   = ((x & 7) >> 1) + ((y & 7) << 2);
        unsigned char sub = subtable[stride * tile_idx * 0 /* embedded in ptr */ + sub_off + /* tile*32 */ tile_idx * 32];
        /* nibble from sub at bit (x&1)*4 */
        int nibble = (sub >> ((x & 1) * 4)) & 0xf;
        return nibble | (attr_nib << 4);
    } else if (view == 1) {
        /* 64-col tilemap */
        int col   = (y >> 3) * 64 + (x >> 3);
        unsigned short tile = tilemap[stride * col];
        int sub_off   = (x & 7) + ((y & 7) << 3);
        unsigned char sub = subtable[/* tile * 64 + sub_off */ (tile << 6) + sub_off];
        return sub;
    }
    return 0;
}
```

**Levers:**
- `lsr #0x1d` for 3-bit stride: `(act[0x29c/4] << 29) >> 29` (unsigned shifts).
- View=0 uses `lsl #0x5` (row * 32), view=1 uses `lsl #0x6` (row * 64).
- The `.word 0x000003ff` constant: write `tile & 0x3ff` or `tile & 0x000003ff` — mwcc will load the constant from the literal pool when it cannot encode it directly.
- Sub-tile pointer arithmetic: `add r0, ip, r0, lsr #0x11` = `subtable + (tile << 22) >> 17 = subtable + tile * 32` (view=0); `add r0, ip, r0, lsr #0x10` = `subtable + tile * 64` (view=1).
- The `mov r0, r4, lsl #0x16 / add r0, ip, r0, lsr #0x11` idiom is mwcc's way to express `subtable + tile_idx * 32` without a multiply — write `subtable + tile_idx * 32`.
