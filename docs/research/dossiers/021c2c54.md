## Struct context

Takes `(int player, int key)`. Inserts `key` into the extra-slot list for `player` at the sorted position determined by a compound key extracted from each existing entry word: `byte6 = (word<<2)>>24`, `sign = (word<<18)>>31`, `sort_key = byte6*2 + sign`. Walks backwards from `count-1` to find first entry with sort_key matching `key`. On match: calls `func_ov002_021b91d0` to swap entry at `r4` with entry at `count`; shifts remaining entries up via `func_ov002_021b91c4`.

**Why similar:** func_ov002_021b91c4 is the word-copy helper used in the shift loop of this function, confirming the `*dst = *src` pattern used in the extra-slot list maintenance.
**Key lesson:** The sort-key extraction `byte6*2 + sign` maps to `mov r0,r0,lsl#1; add r0,r0,r3,lsr#31` — `byte6<<1 + sign`. The two shift pairs are `(word<<2)>>24` for the byte and `(word<<18)>>31` for the sign bit. Mwcc generates a separate `add r0,r0,r3,lsr#0x1f` (shift in add operand) rather than an explicit multiply by 2 — write `byte6*2 + sign` not `byte6<<1 | sign` to get the add form.
