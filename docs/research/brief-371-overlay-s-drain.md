**Brief:** 371 (decomper). Now that the `.s`/GLOBAL_ASM lane runs on Windows
(brief 369, PR #893), drain the **overlay** reg-alloc backlog through it:
`asm_escape --whole-function` each TIER-3 (reg-alloc plateau) target from the

## Result

## Deliverable 1 — robustness: turnkey binutils resolution

`asm_escape.py` called `arm-none-eabi-objdump` by bare name, so it only worked if
`tools/arm-none-eabi/bin` happened to be on `PATH` — otherwise it died with
FileNotFoundError under native `CreateProcess` (the same class of bug the
absolute `_MWCC` / `_MWASM` paths already avoid). Fixed with a `_binutil()`
resolver: it returns the absolute path to the downloaded
`tools/arm-none-eabi/bin/<tool><exe>`, and falls back to the bare name when the
download isn't present (so a system-installed toolchain on Linux/macOS still
works). `_OBJDUMP` flows through it. Confirmed: with `arm-none-eabi-objdump`
**not** on `PATH`, the lane runs end-to-end.

### Sub-finding — xPack `objdump` conditional-ld/st ordering (`to_mwasm`)

`func_ov000_021ab850` failed to assemble: `Undefined macro or opcode: strhls`.
The GNU/xPack `objdump` prints conditional load/stores as `<op><size><cond>`
(`strhls` = str + halfword + `ls`), but mwasmarm — and every Mac-generated ship
already in the tree (`streqh`, `strltb`, `ldrneh`) — want `<op><cond><size>`
(`strlsh`). This is a binutils-version difference: the original ships were
disassembled on a Mac Homebrew `objdump`; the xPack Windows build emits the other
order. Fixed `to_mwasm()` to reorder the size suffix after the condition; forms
with only one of {size, cond} (`strh`, `streq`) pass through unchanged. Unit test
added (`test_conditional_ldst_size_reorder`); 23/23 pass. After the fix
`func_ov000_021ab850` regenerated byte-identical first try.

## Scope: TIER 3 only — TIER 2b left for the permuter

I deliberately did **not** `.s` the TIER-2b crack-once-propagate families
(script-step dispatcher ×4, switch-discriminant ×6, post-blx increment ×3, BGCNT
bit-pack ×4). Those are reg-alloc-CLASS but flagged for a *bounded permuter
probe* first: a single crack reclassifies a whole family to real C, and `.s`-ing
the canonical instance now would forfeit that leverage. They stay queued for the
permuter (WSL, or the ½-day `import.py` patch — brief 369).
