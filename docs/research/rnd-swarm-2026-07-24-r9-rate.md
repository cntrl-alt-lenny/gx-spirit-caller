## The rate decomposition

Per shipped function, effort splits across five stages — and only one of them is the binding constraint:

| Stage | Cost per unit | Bottleneck? |
|---|---|---|
| **Targeting** — pick the next candidate | ~0 (size-first sort) but *mis-targets* as cells drain | miscalibrated, not slow |
| **Agent crack** — reason + iterate to byte-match | N reasoning rounds; each near-miss iteration = 1 manual fastmatch/capstone localization round-trip (~30–50/day across 4 lanes) | secondary sink |
| **Agent self-verify** | ~0.3s via objdiff-cli direct-file mode (post_edit hook) | not a bottleneck |
| **Queue idle** — lane waits for brain restock | lanes drain to QUEUE-EMPTY between rounds | secondary sink |
| **Brain merge-round** — ~10–15 serial hand-merges + conflict-resolve + index-regen + lint-fix + one 3-region gate | ~222s gate (**always-dirty**) + tens of min hand-loop, **once per batched round, human-driven** | **THE bottleneck** |

**The single biggest bottleneck is brain serialization, and the decisive evidence is the overnight dead zone.** On 2026-07-23→24 the brain was idle 20:56→07:08 (~10.2h), yet the 8 ship-branches merged in the 07:08 batch had their tips committed the *prior evening*, 21:20–22:53 — finished, ready-to-merge agent work sat idle ~8h waiting for the human brain operator. That is ~1 lost round/night ≈ 25–35 unshipped functions, ~33% of the day at zero throughput. The 4 agents are not the constraint; the one serial human-driven merge-round is. Two independent taxes compound it: the 3-region `ninja sha1` gate is *always dirty* (a phantom `delink.yaml` output forces a full ~222s relink even on zero-change rounds), and ~40% of the last-300 brain commits (86 hand-merges + index-regen + lint + queue hygiene) are bookkeeping rather than ships.

**2. Precondition that makes an unattended overnight round cheap. *Effort:* hours.

**3. `tools/integrate.py` — one-command merge-round driver.** *(tooling, A)* The brain hand-drives every round: ~10–15 serial `git merge origin/... into brain/integ`, interleaved conflict resolution, index regen, lint fix, one gate3, push — no orchestrator above gate3.py. 86 hand-merge commits + ~40% bookkeeping in last-300; the ~8h idle-ready overnight dead zone proves the brain, not agents, is the bottleneck. *Action:* merge each ref with typed resolvers else abort-and-continue; then `check_delink_dupes` → the 3 index generators → `markdownlint-cli2 --fix` → gate3.py; print a PASS/FAIL/CONFLICT table; idempotent/resumable. *Mechanism:* a round costs minutes not tens of minutes, and an unattended/cron overnight round becomes feasible → reclaims ~1 lost round/night (~25–35 ships). Composes levers 6/domain-B hygiene into one step. *Effort:* days.

**4. **5. Drain-aware per-(module,band) empirical-rate scoring.** *(technique, A)* Static size-band rates decay as cells drain, so a global size-first sort mis-targets: ov002 ≤128B shipped 87% (20/23) but its easy wins are already swept (briefs 650/654, 39 ledger rows), while main ≤128B — 873 candidates, the largest untouched high-yield block — is barely sampled (10 rows, 30%). match_pct is anti-informative (r8), so per-cell empirical rate is the only trustworthy targeting signal. *Action:* score each cell = `shipped/total_attempted` filtered to (module,band) from attempts.tsv with a Bayesian prior (+2/+3 pseudocounts) for thin cells, recomputed every run; emit main ≤128 and ov002 ≤128 as separate ranked streams. (Denominator must count all non-ship park classes, not literal "parked".) wall_aware_headroom.py already has the selectors and never reads the result column. *Mechanism:* keeps the scarce high-EV ≤128 attempt budget on the under-served high-yield cell (main's 873) and self-throttles cells as they drain — sustains blended ship-rate past the first wave. *Effort:* day.

**6. `work_queue.py dedup` + `restock` subcommands.** *(tooling, A)* Queue hygiene+restock is the largest hand-bookkeeping category, yet the tool exposes only next/status/list/done/park; the brain hand-unions restock text (repeatedly breaking MD022) and hand-clears stale CLAIMED/PARKED. ## Won't move the rate (recorded)

- **A per-ship incremental ROM gate (measured negative).** The 74s chain is a *merge-time* cost, not per-ship; agents already self-verify byte-identity via objdiff direct-file mode in ~0.3s (post_edit.py), and a single-module rebuild cannot produce the cross-region whole-image identity the full gate exists for. Merge timestamps cluster (8/3/3), so the gate is amortized once per batched round. Reallocate that effort to the brain-gate/restock levers; reuse `gate3.py --scope eur` if a local smoke is ever wanted.
- **A standalone `localize.py` (reclassified BUILD → display).** The offset→line join the ledger scoped is *already performed* — by objdiff-cli per-instruction (lever 4) and by `objdump --dwarf=decodedline` (38ms) — because `-sym on` is default CC_FLAGS. Re-deriving it in a new tool is duplicated work; ship the same iteration-time win as a ~50-line display change in the hook + fastmatch and close the build task.
- **A global size-first candidate sort.** Looks like disciplined targeting but mis-fires once cells drain: ov002 ≤128's historical 87% is already spent while main ≤128's 873-candidate cell stays high-yield and under-served. Only the drain-aware per-cell score (lever 5) tracks the live rate; a frozen size constant does not.
