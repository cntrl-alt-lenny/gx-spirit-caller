# func_0202c948 (main, HIGH class A, 88B)

**Recipe:** field stores + conditional-move select (movne/moveq) then store
**Risk:** if/else may branch instead of movne/moveq; base reload-after-call and store order 0x4/0x18/0x14 must hold

## SHA1 pitfall

**Primary mode:** Mode E - Visible instruction byte diff (verify.py catches directly)

**Secondary mode:** Mode A — named pool global `data_0219ad48`; address wildcarded by verify.py; only ninja report catches a wrong value

**Secondary mode:** Mode B — callees `func_0202cc8c`, `func_0202c9c0`, `func_02034784`; BL relocs wildcarded by verify.py; only ninja report catches wrong values

**Why this recipe hits this mode:** The conditional `*(data_0219ad48 + 0x0c) = func_02034784() ? 0xf0 : 0x100` must compile to `movne r1,#0xf0` / `moveq r1,#0x100` conditional-move pair; if mwcc emits a branch-taken/branch-not-taken pair the instruction bytes differ visibly. Store ordering (0x4 then 0x18 then 0x14) and base-reload-after-call are also visible constraints.

**Pool words to verify against orig .s:**
- `data_0219ad48` — struct base; `.word` pool entry reloaded after each BL call for every field store

**Verify gate:**
1. python tools/verify.py src/main/func_0202c948.c func_0202c948 --cc all (catches movne/moveq vs branch and store order)
2. ninja report (resolves pool word for data_0219ad48 and BL relocs; catches Mode A/B mismatches)
3. ninja sha1 (final gate)

## Reshape hints

- Use a ternary `*(int*)(data_0219ad48 + 0x0c) = func_02034784() != 0 ? 0xf0 : 0x100` to bias mwcc toward emitting movne/moveq rather than a branch pair
- Reload `data_0219ad48` base into a local pointer after each BL call to match the orig base-reload-after-call pattern
- Write the three post-call stores in exact .s order: offset 0x4 (`= -1`) first, then 0x18 (`= 0`), then 0x14 (`= 0`)
- If movne/moveq does not appear, try `int sel = func_02034784(); *(int*)(data_0219ad48 + 0x0c) = sel ? 0xf0 : 0x100;` to give the compiler a cleaner conditional-move opportunity
- Cross-check `data_0219ad48` pool `.word` value against `symbols.txt` before ninja report to rule out Mode A early
