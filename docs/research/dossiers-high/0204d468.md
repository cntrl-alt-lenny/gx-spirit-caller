# func_0204d468 (main, HIGH class A, 76B)

**Recipe:** Copy32 wrapper + unsigned guard (n>2) + two field stores
**Risk:** Copy32 arg order (dst,src,n); n+2 unsigned compare; f348/f34c/0x350 offsets
**Confidence:** med

## SHA1 pitfall

**Primary mode:** Mode B — indirect callees wildcarded by verify.py BL reloc

**Secondary mode:** Mode E — visible instruction byte diff; unsigned vs signed branch (bhi vs bgt) caught by verify.py

**Why this recipe hits this mode:** `Copy32` and `func_020498f0` are both extern BL/BLX targets whose addresses are wildcarded by verify.py; a wrong callee is silent until ninja report. Independently, the unsigned guard `(unsigned)n > 2` must emit `bhi` not `bgt` — a signed compare would produce a visible instruction diff caught by verify.py.

**Pool words to verify against orig .s:**
No named-symbol pool words — raw hex constants only (compared directly by verify.py).

**Verify gate sequence:**
1. verify.py src/main/func_0204d468.c func_0204d468 [--cc all] — instruction bytes (catches Mode E unsigned branch mismatch)
2. ninja report — resolves BL targets for Copy32 and func_020498f0; catches Mode B callee mismatches before sha1
3. ninja sha1 — final gate

## Reshape hints

- The unsigned guard must be exactly `(unsigned)n > 2` to force a `bhi` (unsigned branch-higher) instruction; dropping the cast to `unsigned` produces `bgt` (signed) and a visible diff.
- `Copy32` argument order is `(dst, src, nbytes)` — destination is `(char*)a1 + 4` (input buffer past the count word), source is `(char*)func_020498f0() + 0x350` (global struct field).
- Byte count to Copy32 must be `(n - 2) << 2`: the two-element header (a1[0] itself) is excluded, and word elements are 4 bytes each.
- The two field stores (`f348 = n - 1`, `f34c = a0`) go after the Copy32 call and each must reload the accessor via BL — do not cache the pointer across stores.
- Cross-check the addresses of `Copy32` and `func_020498f0` against the .s BL instructions to confirm no callee swap.
