# func_0202bb88 (main, HIGH class A, 64B)

**Recipe:** counted byte-clear loop, guard, no calls
**Risk:** loop may index [base+0xf6c, i] vs the asm's pointer-walk (add r3,#1); ldr [r0] needs int-typed count

## SHA1 pitfall

**Primary mode:** Mode A - Wrong pool word value (HIDDEN from verify.py; only ninja report catches)

**Secondary mode:** Mode E - Visible instruction byte diff if loop uses index recompute vs the .s pointer-walk (`add r3,#1`)

**Why this recipe hits this mode:** Both `data_020b4768` (the count word) and `data_02104f4c` (the byte array base) are loaded via pool `.word` entries whose values are wildcarded in verify.py's reloc pass; wrong symbol addresses pass verify.py but fail ninja report and sha1. Additionally, if mwcc emits `data_02104f4c[i + 0xf6c]` as an index recompute rather than a pointer walk, the loop body instruction sequence changes visibly.

**Pool words to verify against orig .s:**
- `data_020b4768` — count-load base; `.word` pool entry read as `*(int*)data_020b4768` for the iteration count
- `data_02104f4c` — byte array base; `.word` pool entry used as `&data_02104f4c[i + 0xf6c]` for the bit-clear `ldrb`/`strb`

**Verify gate:**
1. python tools/verify.py src/main/func_0202bb88.c func_0202bb88 --cc all (catches loop form and bit-clear encoding)
2. ninja report (resolves pool words for data_020b4768 and data_02104f4c; catches Mode A mismatch)
3. ninja sha1 (final gate)

## Reshape hints

- Declare `count` as `int` (not `unsigned int`) so the count LDR emits a plain signed word load matching the orig
- Match the loop form to the .s: if orig uses `add r3,#1` on a byte pointer, introduce `unsigned char *p = &data_02104f4c[0xf6c]` and increment `p++` in the loop rather than using `data_02104f4c[i + 0xf6c]`
- The `&= ~8` bit-clear must use `ldrb`/`strb`; declare `data_02104f4c` as `unsigned char[]` not `int[]`
- Cross-check `data_020b4768` address in pool against `symbols.txt` to confirm the count variable is at the right address
- Cross-check `data_02104f4c` base address against the .s `.word` entry to rule out Mode A pool mismatch before ninja sha1
