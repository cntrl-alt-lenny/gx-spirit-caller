# func_0202ce24 (main, HIGH class A, 84B)

**Recipe:** bind-once guard chain + two tail calls
**Risk:** func_02037208 arg1=-1 may materialize via mov/mvn not sub-from-0x35; reload-vs-bind of global may differ

## SHA1 pitfall

**Primary mode:** Mode A - Wrong pool word value (HIDDEN from verify.py; only ninja report catches)

**Secondary mode:** Mode E - Visible instruction byte diff if `-1` materializes as `sub r1,r0,#0x36` (derived from `r0=0x35`) rather than `mvn r1,#0`

**Secondary mode:** Mode B — callees `func_0202d194` and `func_02037208`; BL relocs wildcarded by verify.py; only ninja report catches wrong values

**Why this recipe hits this mode:** The bind-once pattern loads `data_0219ad48` address once into a register and reuses it across all four guard reads; if the compiler re-loads the pool word between reads the LDR sequence and register usage change, and the wrong pool word address passes verify.py silently. The `-1` argument encoding (sub vs mvn) is a separate visible diff.

**Pool words to verify against orig .s:**
- `data_0219ad48` — struct base; single `.word` pool entry held in a register across all four guard reads (`+0x00`, `+0x24`, `+0x18`, `+0x14`) and the two tail calls

**Verify gate:**
1. python tools/verify.py src/main/func_0202ce24.c func_0202ce24 --cc all (instruction bytes)
2. ninja report (resolves pool word for data_0219ad48 and BL relocs; catches Mode A/B mismatches)
3. ninja sha1 (final gate)

## Reshape hints

- Bind the global to a single local `S_0202ce24 *p = &data_0219ad48` and access all four fields through `p` without re-casting to keep the bind-once register pattern
- Keep guard-chain order exactly: `p->active`, `p->field_24`, `p->field_18`, `p->field_14` — each `ldrne`/`cmpne` in the .s corresponds to one guard in textual order
- For the `-1` argument: check the .s instruction; if it is `sub r1,r0,#0x36` (derived after `mov r0,#0x35`), the literal `func_02037208(0x35, -1, 0, 1)` call should naturally produce this; verify which encoding mwcc chooses
- If mwcc reloads the pool word between guard reads, try extracting `p->active` into an intermediate variable to force an earlier register commit
- Cross-check `data_0219ad48` pool `.word` value against `symbols.txt` before ninja report to rule out Mode A mismatch early
