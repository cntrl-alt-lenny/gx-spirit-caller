# func_02054b00 (main, HIGH class A, 28B)

**Recipe:** predicated conditional store (if/else same global, bic mask)
**Risk:** unused a1 must not perturb regs; & 0x7fffffff must encode as bic not and
**Confidence:** high

## SHA1 pitfall

**Primary mode:** Mode A — hidden pool-word order mismatch; data_020ffbd4 is a named global used as a store target, and its pool slot is wildcarded by verify.py; only ninja report catches a wrong literal address

**Secondary mode:** Mode E — visible instruction diff; `a0 & 0x7fffffff` must emit `bic` (clear bit 31) not `and`; the immediate encoding differs and verify.py will surface this directly

**Why this recipe hits this mode:** `data_020ffbd4` is loaded as a store base for the conditional write. As a named data_ global its pool address is allocated by MWCC; if the pool slot does not match the orig .s, verify.py's instruction-byte comparison wildcards the ldr-literal and the mismatch is invisible until ninja report resolves the reloc. The bic vs and encoding for `0x7fffffff` is a separate, visible-mode risk that verify.py catches directly.

**Pool words to verify against orig .s:**
- `data_020ffbd4` — store base for the conditional int write; used in both the bicne and moveq paths

**Verify gate sequence:**
1. verify.py src/main/func_02054b00.c func_02054b00 [--cc all] — instruction bytes
2. ninja report — resolves pool words; catches Mode A data_020ffbd4 slot mismatch before sha1
3. ninja sha1 — final gate

## Reshape hints

- The mask `a0 & 0x7fffffff` must appear exactly as written — MWCC encodes this as `bic r0, r0, #0x80000000` (clear bit 31); do not factor into a named constant or intermediate variable that might force `and` encoding
- Declare `a1` as `int a1` in the parameter list but do not reference it anywhere in the body — MWCC will assign it to r1 and leave r1 alone, matching the orig; adding any use of a1 perturbs the register schedule
- The if/else writing the same global is the predicated pattern (bicne/moveq str); ensure both branches write `data_020ffbd4` — any restructuring to a ternary or a single-path write may change the predicated instruction encoding
- Use `extern int data_020ffbd4;` (plain int, not pointer) so the store is a single `str` without index; a pointer type might emit an extra ldr step
- Confirm ninja report shows data_020ffbd4 at address 0x020ffbd4 after link — if the address in the pool differs, swap or reorder the extern declaration relative to other globals in the TU
