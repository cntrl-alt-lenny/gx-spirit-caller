**Why this recipe hits this mode:** `data_0219dc84` is the load base for all struct field accesses; its address is emitted as a pool .word and wildcarded by verify.py, so a wrong global address passes instruction comparison silently and only surfaces in `ninja report`.

**Pool words to verify against orig .s:**
- `data_0219dc84` — primary struct base used for all field reads and writes (f0, f2, f8, fc, f10, f14, f18, f1c); every LDR of this global from the pool must match the orig .s .word entry

**Verify gate sequence:**
1. verify.py src/main/func_0204b034.c func_0204b034 [--cc all] — instruction bytes

3. ninja sha1 — final gate
