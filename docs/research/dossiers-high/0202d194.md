# func_0202d194 (main, HIGH class A, 68B)

**Recipe:** linear: ptr-chase, clear-call, two ordered stores
**Risk:** the +0x24e0 may split into adds at 0x2000/0x4e0 in a different order; r1[0x6c] field type guessed
**Confidence:** med

## SHA1 pitfall

**Primary mode:** Mode E — visible instruction byte diff if the 0x24e0 offset splits as `add r1,r0,#0x2000` before `add r0,r0,#0x4e0` vs the reverse order; verify.py catches it

**Secondary mode:** Mode A — named pool global `data_0219ad48`; address wildcarded by verify.py, only ninja report catches a wrong value

**Secondary mode:** Mode B — callees `func_0201d530` and `func_02094504` addresses wildcarded by verify.py BL reloc, only ninja report catches wrong values

**Why this recipe hits this mode:** 0x24e0 is not an ARM rotated immediate, so mwcc must split the offset into two `add` instructions (`+0x4e0` then `+0x2000`); the order of these adds determines which intermediate register holds which value and produces a visible instruction-byte diff. Additionally both post-call stores to `data_0219ad48 + 0x18` then `+ 0x14` must appear in that exact order after the two BLs.

**Pool words to verify against orig .s:**
- `data_0219ad48` — struct base; `.word` pool entry used for the `handle` LDR at `+0x6c` and for both post-call STR stores

**Verify gate sequence:**
1. verify.py src/main/func_0202d194.c func_0202d194 [--cc all] — instruction diff is visible directly (add-split order, store sequence)
2. ninja report — resolves pool word for data_0219ad48 and BL relocs for func_0201d530 and func_02094504; catches Mode A/B mismatches before sha1
3. ninja sha1 — final gate

## Reshape hints

- Write the end-pointer as `(base + 0x4e0) + 0x2000` explicitly to steer the add-split to match the .s (`add r0,r0,#0x4e0` first, then the 0x2000 add for `end`)
- Keep the two stores in exact .s order: `*(+0x18) = arg0` before `*(+0x14) = 1`; reversing them produces a visible STR-sequence diff
- Reload the `data_0219ad48` base after the two BLs (it is clobbered across the calls); do not attempt to cache it in a caller-saved register across BLs
- Declare `data_0219ad48.handle` at `+0x6c` as `void *` so `func_0201d530` receives a pointer-typed arg without an extra cast instruction
- Cross-check `data_0219ad48` pool `.word` value against `symbols.txt` before ninja report to rule out Mode A early
